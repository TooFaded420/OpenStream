# OpenClaw + Stream Deck Use Cases

## Core Use Cases

### 1. Developer Workflow
**User:** Software developer working on multiple projects
**Setup:**
- PC with OpenClaw (main workstation)
- Mac Mini with OpenClaw (secondary/testing)
- Stream Deck XL (32 keys)

**Buttons:**
- Spawn coding agent for review
- Toggle TTS for voice alerts
- Check build status
- Deploy to staging
- Debug helper
- Git commit helper

**Workflow:**
1. Press "Spawn Code Agent" → AI reviews current file
2. Results appear in Telegram
3. Press "Debug Help" for error assistance
4. Press "Deploy" → triggers CI/CD

**Value:** Hands-free coding assistance without leaving IDE

---

### 2. Content Creator / Streamer
**User:** Twitch/YouTube streamer
**Setup:**
- Stream Deck Plus (dials for audio)
- OpenClaw for AI co-host
- OBS integration

**Buttons:**
- Toggle TTS (AI speaks to chat)
- Spawn "Chat Moderator" agent
- Generate poll questions
- Summarize stream topics
- Fact-check in real-time

**Workflow:**
1. Press "AI Co-host" → AI joins voice chat
2. Viewers ask questions → AI answers via TTS
3. Press "Summarize" → AI posts stream summary
4. Press "Poll" → AI generates engaging questions

**Value:** AI co-host reduces creator workload

## Advanced Use Cases

### 9. Multi-Gateway Control (Your Setup!)
**User:** Developer with PC + Mac Mini
**Setup:**
- PC: Main workstation
- Mac: Testing environment
- Same network, 2 OpenClaw instances

**Buttons:**
- Use PC gateway
- Use Mac gateway
- Use best (auto-switch)
- Broadcast to both
- Sync sessions

**Workflow:**
1. Coding on PC → spawn agent
2. Testing on Mac → switch gateway
3. Deploy → broadcast to both
4. Sync → keep contexts aligned

**Value:** Seamless multi-device AI control
